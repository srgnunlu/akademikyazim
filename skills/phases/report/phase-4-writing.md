# Phase 4 — Report Writing
# Aşama 4 — Rapor Yazımı

## Gate Rule / Geçiş Kuralı

**EN:** findings_and_recommendations.md must be finalized before writing begins. Writing restructures and communicates — it does not generate new analysis. If new analysis is needed, return to Phase 3.

**TR:** Yazımdan önce findings_and_recommendations.md kesinleştirilmelidir. Yazım yeniden yapılandırır ve iletişim kurar — yeni analiz üretmez. Yeni analiz gerekiyorsa Aşama 3'e dönün.

---

## Guided Writing Mode / Rehberli Yazım Modu

Bu fazda `skills/core/guided-writing-mode.md` protokolü aktiftir:
- Her bölüm/paragraf için A/B seçenekleri üretilir
- Her seçenek kaynak kanıtlarıyla desteklenir (📚)
- Akademik Yazım Notu yapıyı ve normu açıklar (📖)
- Kullanıcı seçer, birleştirir veya yeniden yönlendirir
- Demir Kural 1: Kaynak olmadan yazım YOK

In this phase, the `skills/core/guided-writing-mode.md` protocol is active:
- A/B draft options are generated for each section/paragraph
- Each option is backed by source evidence (📚)
- Academic Writing Note explains structure and norms (📖)
- User selects, merges, or redirects
- Iron Rule 1: No writing without source

---

## 1. Writing Order / Yazım Sırası

**EN:** Write sections in this order. The logic inverts academic convention: readers need the bottom line first.

**TR:** Bölümleri bu sırayla yazın. Mantık, akademik sözleşmeyi tersine çevirir: okuyucular önce sonucu ihtiyaç duyar.

```
1. Executive Summary (final update) / Yönetici Özeti (nihai güncelleme)
2. Recommendations / Öneriler
3. Key Findings / Temel Bulgular
4. Methods (evidence base) / Yöntemler (kanıt tabanı)
5. Appendices / Ekler
```

**Why this order / Bu sıra neden:**
- A policymaker who reads only the Executive Summary must be fully informed
- A policymaker who reads further must find Recommendations before detailed evidence
- Methods and Appendices are for accountability and specialist readers

---

## 2. Executive Summary (Final) / Yönetici Özeti (Nihai)

**EN:** The executive summary stands alone. A reader who reads nothing else must understand: what was investigated, what was found, and what is recommended. Maximum: one page. No jargon. No footnotes. Active voice.

**TR:** Yönetici özeti bağımsız olarak durur. Başka hiçbir şey okumayan bir okuyucu şunları anlamalıdır: neyin araştırıldığı, neyin bulunduğu ve neyin önerildiği. Maksimum: bir sayfa. Jargon yok. Dipnot yok. Etken çatı.

```
Structure / Yapı:
Paragraph 1 — Context (2-3 sentences) / Bağlam
Paragraph 2 — Key Findings (3-5 bullets) / Temel Bulgular
Paragraph 3 — Recommendations (numbered list) / Öneriler
Paragraph 4 — Scope/limitations note (1-2 sentences) / Kapsam/sınırlılıklar
```

---

## 3. Language & Accessibility / Dil ve Erişilebilirlik

**EN:** Match language complexity to the audience defined in Phase 0. Practical rules:

**TR:** Dil karmaşıklığını Aşama 0'da tanımlanan hedef kitleyle eşleştirin. Pratik kurallar:

- Define every technical term the first time it appears / Her teknik terimi ilk geçtiğinde tanımlayın
- Sentences under 25 words where possible / Mümkün olduğunda 25 kelimenin altında cümleler
- One idea per paragraph / Paragraf başına bir fikir
- Active voice preferred / Etken çatı tercih edilir
- Spell out acronyms first use / Kısaltmaları ilk kullanımda açın
- Avoid academic hedging for non-academic audiences / Akademik olmayan kitleler için akademik muğlaklıktan kaçının

---

## 4. Visual Elements / Görsel Öğeler

**EN:** Reports differ from academic papers in their encouragement of visual communication. For each finding or data point, ask: could a table or chart communicate this more clearly than prose?

**TR:** Raporlar akademik makalelerden görsel iletişimi teşvik etmesiyle ayrılır. Her bulgu veya veri noktası için sorun: bir tablo veya grafik bunu proza göre daha net iletebilir mi?

**Visual element checklist / Görsel öğe kontrol listesi:**
- [ ] Summary table of all recommendations / Tüm önerilerin özet tablosu
- [ ] Timeline or priority matrix for recommendations / Öneriler için zaman çizelgesi veya öncelik matrisi
- [ ] Data visualizations for key statistics / Temel istatistikler için veri görselleştirmeleri
- [ ] Comparison tables where multiple options are discussed / Birden fazla seçeneğin tartışıldığı karşılaştırma tabloları

---

## 5. Source Tracing (Iron Rules in Report Context) / Kaynak İzleme

**EN:** Every factual claim in the final report must trace back to a source in evidence_inventory.md. Use citation style appropriate to the commissioner's requirements. Parenthetical in-text citations (Author, Year) or numbered footnotes — not endnotes, which are inaccessible.

**TR:** Nihai rapordaki her olgusal iddia, evidence_inventory.md'deki bir kaynağa izlenebilir olmalıdır. Komisyoncunun gereksinimlerine uygun atıf stilini kullanın. Metiniçi parantez atıflar (Yazar, Yıl) veya numaralı dipnotlar — erişilemeyen son notlar değil.

---

## 6. Commissioner Review Cycle / Komisyoncu İnceleme Döngüsü

**EN:** Build in review cycles. Reports are rarely finalized in one draft. Plan for:

**TR:** İnceleme döngüleri planlayın. Raporlar nadiren tek taslakta kesinleşir. Planlayın:

```
Draft 1 / Taslak 1 → Commissioner review / Komisyoncu incelemesi
Draft 2 / Taslak 2 → Fact-check + legal review if needed / Gerçek kontrol + gerekirse hukuki inceleme
Final / Nihai → Sign-off / Onay
```

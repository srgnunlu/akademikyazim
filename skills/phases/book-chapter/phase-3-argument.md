# Phase 3 — Chapter Argument
# Aşama 3 — Bölüm Argümanı

## Gate Rule / Geçiş Kuralı

**EN:** The chapter must have ONE argument. If you cannot state the chapter's argument in a single sentence, the argument is not yet clear enough to write. This does not mean simple — a single argument can be complex and nuanced. It means unified.

**TR:** Bölümün BİR argümanı olmalıdır. Bölümün argümanını tek cümlede ifade edemiyorsanız, argüman henüz yazmak için yeterince açık değildir. Bu basit anlamına gelmez — tek bir argüman karmaşık ve nüanslı olabilir. Birleşik anlamına gelir.

---

## 1. The Chapter's Single Argument / Bölümün Tek Argümanı

**EN:** Write the chapter's central argument in one sentence. This is not the topic — it is the claim. "This chapter examines X" is a topic statement. "This chapter argues that X because Y" is an argument.

**TR:** Bölümün merkezi argümanını tek cümlede yazın. Bu konu değildir — iddiadır. "Bu bölüm X'i inceliyor" bir konu ifadesidir. "Bu bölüm Y nedeniyle X'i savunuyor" bir argümandır.

```
Chapter argument (one sentence) / Bölüm argümanı (tek cümle):
[This chapter argues that ___ because ___ , which means ___.]
[Bu bölüm ___ nedeniyle ___'ı savunuyor; bu da ___ anlamına geliyor.]
```

---

## 2. Sub-Argument Structure / Alt Argüman Yapısı

**EN:** How does the chapter's argument function as a sub-argument of the book's thesis? Map the relationship explicitly:

**TR:** Bölümün argümanı kitabın tezinin bir alt argümanı olarak nasıl işlev görüyor? İlişkiyi açıkça eşleştirin:

```
Book's central argument / Kitabın merkezi argümanı:
[from alignment_brief.md]

Chapter's contribution to it / Bölümün katkısı:
"This chapter supports/extends/complicates/provides evidence for the book's argument
by [specific contribution]."

"Bu bölüm [özgül katkı] yaparak kitabın argümanını destekliyor/genişletiyor/
karmaşıklaştırıyor/için kanıt sağlıyor."
```

---

## 3. Argument Map / Argüman Haritası

**EN:** Map every claim in the chapter and its supporting evidence. This is the architecture that the writing phase will build out. Every claim must have evidence; evidence without a claim is not an argument.

**TR:** Bölümdeki her iddiayı ve destekleyici kanıtını eşleştirin. Bu, yazım aşamasının geliştireceği mimaridir. Her iddianın kanıtı olmalıdır; iddianın olmadığı kanıt argüman değildir.

```markdown
## Chapter Argument Map / Bölüm Argüman Haritası

**Central argument / Merkezi argüman:** [one sentence]

### Sub-claim 1 / Alt iddia 1
**Claim / İddia:** [specific claim]
**Evidence / Kanıt:** [source(s) from reading notes]
**Role in overall argument / Genel argümandaki rol:** [how does this support the central argument?]

### Sub-claim 2 / Alt iddia 2
**Claim / İddia:**
**Evidence / Kanıt:**
**Role / Rol:**

### Counter-argument addressed / Ele alınan karşı argüman
**Counter-claim / Karşı iddia:**
**Response / Yanıt:**
**Evidence for response / Yanıt için kanıt:**
```

---

## 4. Connective Tissue to the Book / Kitaba Bağlayıcı Doku

**EN:** The connections to the book's argument cannot be implicit. They must be explicit textual signals in two mandatory locations:

**TR:** Kitabın argümanıyla bağlantılar örtük olamaz. İki zorunlu konumda açık metinsel sinyaller olmalıdır:

**Introduction / Giriş:**
- State the chapter's argument
- Explicitly connect it to the book's central argument
- Acknowledge what previous chapters have established that this chapter builds on

**Conclusion / Sonuç:**
- Restate how the chapter's argument advances the book's thesis
- Signal explicitly to the next chapter or the book's conclusion
- Do not introduce new evidence — consolidate

```
Introduction connective sentence / Giriş bağlayıcı cümle:
"Building on [Chapter N]'s argument that ___, this chapter argues ___."
"[Bölüm N]'nin ___ argümanı üzerine inşa eden bu bölüm ___'ı savunuyor."

Conclusion connective sentence / Sonuç bağlayıcı cümle:
"This chapter's argument that ___ advances the volume's central claim that ___ by ___."
"Bu bölümün ___ argümanı, ___ yaparak cildin merkezi iddiasını ___ geliştiriyor."
```

---

## 5. Structural Outline / Yapısal Ana Hat

**EN:** Convert the argument map into a section outline with word count allocation. This becomes the writing blueprint.

**TR:** Argüman haritasını kelime sayısı tahsisiyle bölüm ana hattına dönüştürün. Bu, yazım planı olur.

```
Total word count / Toplam kelime sayısı: ___

Introduction / Giriş: ___ words
Section 1 [title] / Bölüm 1: ___ words — argues: ___
Section 2 [title] / Bölüm 2: ___ words — argues: ___
Section 3 [title] / Bölüm 3: ___ words — argues: ___
Conclusion / Sonuç: ___ words
```
